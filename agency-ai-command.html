Getting Started with AgencyAI

This guide will help you understand the core concepts and get up to speed quickly.

1. Understanding the Dashboard

The dashboard provides an overview of all your AI agents, their current status, and recent activity. Key metrics include:

  • Active Agents - Number of agents currently running
  • Tasks Today - Total automated tasks completed
  • Campaigns Running - Active marketing campaigns
  • System Uptime - Server reliability metric

2. Managing AI Agents

Each agent has specific responsibilities and can be configured independently:

  • Use the Configure button to adjust agent settings
  • View real-time logs to monitor agent activity
  • Start/stop agents as needed for maintenance

3. Monitoring Workflows

Workflows show how agents work together to complete complex tasks. Green status dots indicate active pipeline stages.

4. Accessing the AI Assistant

Click the blue chat button in the bottom-right corner to ask questions about the system. The assistant can help with troubleshooting and configuration.

Configuring Your AI Agents

Each agent can be customized to match your specific needs and workflows.

Content Creator Configuration

  • Brand Voice: Upload brand guidelines and sample content
  • Content Types: Configure which types to generate (blog, social, ads)
  • Review Process: Set approval workflows before publication
  • API Settings: Adjust Claude API parameters for quality/speed balance

Lead Hunter Configuration

  • Target Criteria: Define your ideal customer profile
  • Data Sources: Select platforms to scrape (LinkedIn, Google Maps, etc.)
  • Qualification Rules: Set AI-powered scoring criteria
  • Rate Limits: Configure scraping speed to avoid blocks

SEO Analyzer Configuration

  • Tracked Keywords: Add keywords to monitor rankings
  • Competitors: Define competitor URLs for analysis
  • Audit Frequency: Schedule automatic website audits
  • Reporting: Configure which metrics to track

Email Architect Configuration

  • Email Templates: Create reusable sequence templates
  • Personalization: Set merge tags and dynamic content rules
  • A/B Testing: Configure test variants and success metrics
  • CRM Integration: Connect your CRM for lead sync

Integrating External APIs

AgencyAI supports integration with multiple AI providers and marketing tools.

Supported AI APIs

  • OpenAI: GPT-4 for content generation and analysis
  • Anthropic Claude: Advanced reasoning and long-form content
  • Custom Models: Add your own fine-tuned models

Adding API Keys

Navigate to Settings → API Configuration to add your keys. All keys are encrypted at rest and never logged.

  • OpenAI: Paste your sk-... key
  • Claude: Add your Anthropic API key
  • Rate limits are automatically detected and respected

Marketing Tool Integrations

  • CRM: HubSpot, Salesforce, Pipedrive
  • Email: SendGrid, Mailgun, AWS SES
  • Social: Buffer, Hootsuite, native APIs
  • Analytics: Google Analytics, Mixpanel

Common Issues and Solutions

Agent Won't Start

  • Check server status in the Server Monitor tab
  • Verify API keys are valid (Settings → API Configuration)
  • Review agent logs for error messages
  • Restart the Docker container if needed

Low API Token Limits

  • Monitor usage in the Server Monitor tab
  • Adjust agent frequency to reduce API calls
  • Upgrade your API plan if consistently hitting limits
  • Use caching where possible to reduce redundant calls

Workflow Not Completing

  • Check if all required agents are running
  • Verify integration credentials (CRM, email, etc.)
  • Review workflow logs for failed steps
  • Ensure data format matches expected schema

Performance Issues

  • Check CPU and memory usage in Server Monitor
  • Reduce concurrent agent tasks if overloaded
  • Clear old logs and data to free disk space
  • Consider upgrading server resources if consistently maxed out

Getting Help

For issues not covered here:

  • Use the AI Assistant chat for instant help
  • Check server logs: /var/log/openclaw/
  • Contact support with log excerpts and error details

Keeping Your System Secure

Access Control

  • Use strong passwords for all accounts
  • Enable two-factor authentication where available
  • Limit SSH access to specific IP addresses
  • Regularly review user permissions and revoke unused access

API Key Management

  • Never commit API keys to version control
  • Rotate keys regularly (every 90 days recommended)
  • Use separate keys for dev/staging/production
  • Monitor API usage for anomalies

Server Security

  • Keep Ubuntu and all packages up to date
  • Enable automatic security updates
  • Use firewall rules to restrict ports
  • Regular backups (automated daily backups enabled)

Data Protection

  • All data encrypted at rest and in transit
  • Regular database backups (retained for 30 days)
  • GDPR-compliant data handling
  • Client data isolated per workspace
