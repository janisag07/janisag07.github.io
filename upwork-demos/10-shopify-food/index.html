🍽️ Weekly Menu Automation

Shopify solution for automatic menu scheduling

❌ The Problem

Food businesses with rotating weekly menus currently have to manually publish/unpublish products every week. This means:

  • Waking up early Sunday to update the menu
  • Risk of forgetting and selling unavailable items
  • No way to schedule changes in advance
  • Staff time wasted on repetitive manual work

✅ The Solution

An automated scheduling system that publishes/unpublishes menu items based on predefined schedules. Set it once, and it handles the weekly rotation automatically — no manual intervention needed.

How It Works

1

Tag Products by Week

Add tags to products indicating which week(s) they're available: week-1, week-2, etc. Products can have multiple week tags if they're available multiple weeks.

2

Calculate Current Week

System automatically determines the current week number of the year (1-52). This ensures the correct menu is always shown without manual input.

3

Scheduled Publish/Unpublish

Cron job runs at configured times (e.g., Sunday 6AM) to publish products matching current week and unpublish products from previous week.

4

Admin Dashboard

Simple interface to see what's currently active, preview next week's menu, and override schedules if needed.

// Simplified scheduling logic (n8n / Node.js)

async function updateWeeklyMenu() {
  const currentWeek = getISOWeek(new Date());
  const previousWeek = currentWeek === 1 ? 52 : currentWeek - 1;
  
  // Unpublish last week's products
  await shopify.products.updateByTag(
    `week-${previousWeek}`,
    { status: 'draft' }
  );
  
  // Publish this week's products
  await shopify.products.updateByTag(
    `week-${currentWeek}`,
    { status: 'active' }
  );
  
  // Log & notify
  console.log(`Menu updated: Week ${currentWeek} now active`);
  await sendSlackNotification(`✅ Week ${currentWeek} menu is live!`);
}
