Automation Tasks

1 Two-Way Deal Name Sync
~6–8 hours

When a project name changes in either PaintScout or CompanyCam, the other system updates automatically. Both directions.

PaintScout → CompanyCam
  1. Deal name updated in PaintScout
  2. Zapier catches the "Updated Deal" trigger
  3. Passes data to Make.com via webhook
  4. Make.com looks up CompanyCam project ID in Google Sheets
  5. HTTP PATCH to CompanyCam API → name updated
CompanyCam → PaintScout
  1. Project name updated in CompanyCam
  2. CompanyCam webhook fires to Make.com
  3. Make.com looks up PaintScout deal ID in Google Sheets
  4. Triggers Zapier action to update PaintScout deal
🛡 Loop Prevention:

Each sync writes a transaction ID + timestamp to Google Sheets. Before syncing, check if the same update came from the other direction within 60 seconds. If yes → skip. Breaks infinite ping-pong.

2 Invoice Work Order Link Sync
~4–5 hours

When an invoice is created in PaintScout, automatically link it to the corresponding CompanyCam work order. One-time per invoice, permanent reference.

PaintScout → CompanyCam (one-way)
  1. New invoice created in PaintScout
  2. Zapier catches "New Invoice" trigger
  3. Make.com looks up matching work order in Google Sheets
  4. HTTP POST to CompanyCam API → attaches invoice link
  5. Marks invoice as synced in Google Sheets (idempotency)
🛡 Duplicate Prevention:

One-directional only. Google Sheets tracks synced invoice IDs. If already synced → skip. No reverse trigger needed.

3 Pipeline Stage Two-Way Label Sync
~7–9 hours

Add new pipeline stages and sync labels bidirectionally. When a deal moves stages in PaintScout, the CompanyCam label updates — and vice versa.

PaintScout → CompanyCam
  1. Deal moves to new stage in PaintScout
  2. Zapier catches stage change trigger
  3. Make.com maps PaintScout stage → CompanyCam label via Google Sheets lookup table
  4. HTTP PATCH to CompanyCam API → label updated
CompanyCam → PaintScout
  1. Label changed in CompanyCam
  2. Webhook to Make.com → reverse lookup in Google Sheets
  3. Triggers Zapier action → PaintScout deal stage updated
🛡 Loop Prevention:

Google Sheets mapping table stores last_sync_source per project. Before syncing, check: did this stage value come from our own sync? If yes → skip. Plus 90-second cooldown window to absorb delayed webhooks.

4 Salesperson Reassignment Sync
~5–6 hours

When a salesperson is reassigned in PaintScout, automatically update the assigned user in CompanyCam. Keeps your team aligned across platforms.

PaintScout → CompanyCam (one-way)
  1. Deal owner changes in PaintScout
  2. Zapier catches "Updated Deal" trigger with owner field change
  3. Make.com maps PaintScout user → CompanyCam user via Google Sheets
  4. Verifies new owner ≠ current CompanyCam owner (skip if same)
  5. HTTP PATCH to CompanyCam API → assigned user updated
🛡 Duplicate Prevention:

One-directional. Check current CompanyCam owner before making the API call. If already matching → skip entirely. No unnecessary API calls.
