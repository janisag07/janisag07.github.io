1 Two-Way Deal Name Sync

Synchronize deal names bidirectionally between PaintScout and CompanyCam. When a deal name changes in either system, the other system updates automatically within minutes.

PaintScout → CompanyCam
Zapier: Deal Updated Trigger
Catches deal name changes in PaintScout
↓
Filter: Name Changed?
Only proceed if deal name actually changed
↓
Google Sheets: Log Update
Record timestamp + source = "PaintScout"
↓
Make: Update CompanyCam Project
HTTP PUT to CompanyCam API with new name
CompanyCam → PaintScout
Make: Webhook from CompanyCam
Project update webhook fires on name change
↓
Filter: Name Changed?
Verify name field actually changed
↓
Google Sheets: Log Update
Record timestamp + source = "CompanyCam"
↓
Zapier: Update PaintScout Deal
Built-in Zapier action updates deal name
⚠️ Loop Prevention

Google Sheets tracks last update source + timestamp. If update comes from system A and would sync back to A within 5 minutes, abort. Prevents infinite ping-pong updates.

⏱ Estimated Time: 3 days (design, implementation, testing)
Zapier Make.com Google Sheets CompanyCam API

2 Invoice Work Order Link Sync

When an invoice is created in PaintScout, automatically attach a direct link to the corresponding CompanyCam work order. One-time sync per invoice, idempotent and error-resistant.

Invoice Created → Link Added
1. Zapier: Invoice Created Trigger
Fires when new invoice is generated in PaintScout
↓
2. Lookup CompanyCam Project ID
Match deal → project via Google Sheets mapping table
↓
3. Check: Already Linked?
Query Google Sheets: Has this invoice been linked before?
↓
4. Generate Work Order URL
Format: https://app.companycam.com/projects/{project_id}
↓
5. Make: Update PaintScout Invoice
Add work order link to invoice notes/custom field
↓
6. Log to Google Sheets
Record invoice ID + project ID + timestamp
⚠️ Loop Prevention

Invoice-to-project mapping stored in Google Sheets. Before adding link, check if invoice ID already exists in log. If yes, skip (idempotent). Prevents duplicate links on invoice updates.

⏱ Estimated Time: 2 days (one-way sync, simpler logic)
Zapier Make.com Google Sheets

3 Pipeline Stages — Two-Way Label Sync

Create new pipeline stages in PaintScout and sync deal stage movements bidirectionally with CompanyCam project labels. When a deal moves to a new stage, CompanyCam labels update accordingly, and vice versa.

Stage Mapping Configuration

Google Sheets: Log Update

↓

HTTP POST to add/remove project labels
CompanyCam Label Change → PaintScout Stage

Same timestamp-based source tracking as deal name sync. If stage update originates from PaintScout, don't sync back to PaintScout. 5-minute window prevents bouncing. Mapping table ensures only configured stages trigger syncs.

Zapier Make.com Google Sheets CompanyCam API

4 Salesperson Reassignment Sync

When a deal's salesperson/owner changes in PaintScout, automatically update the assigned user in CompanyCam. One-way sync (PaintScout → CompanyCam) to maintain single source of truth for sales assignments.

Salesperson Change Flow
1. Zapier: Deal Owner Changed Trigger
Fires when salesperson/owner field updates in PaintScout
↓
2. Lookup User Mapping
Google Sheets: PaintScout user → CompanyCam user ID
↓
3. Validate User Exists
Check if mapped CompanyCam user is active
↓
4. Make: Reassign CompanyCam Project
HTTP PATCH to update project's assigned user
↓
5. Log to Google Sheets
Record deal ID + old user + new user + timestamp
↓
6. Error Handling
If user not found, notify admin via Slack/email, don't fail silently
⚠️ Loop Prevention

One-way sync only (PaintScout → CompanyCam), so no loop risk. User mapping table prevents errors. If CompanyCam user is manually changed, it won't sync back — PaintScout remains source of truth for assignments.

⏱ Estimated Time: 2 days (one-way, straightforward mapping)
Zapier Make.com Google Sheets CompanyCam API

End-to-End Testing Strategy

Phase 1: Unit Testing (Per Feature)

  • Test each automation in isolation with test data
  • Verify triggers fire correctly
  • Confirm filters block unwanted executions
  • Validate API calls return expected responses
  • Check Google Sheets logging accuracy

Phase 2: Integration Testing

  • Test all 4 features together with real data clones
  • Simulate rapid changes to test race conditions
  • Verify loop prevention works under stress
  • Test error handling (API timeouts, invalid data)
  • Confirm sync timing (usually <2 minutes end-to-end)

Phase 3: Edge Cases & Scenarios

  • Deal name changed in both systems simultaneously
  • Invoice created for deal without CompanyCam project
  • Pipeline stage changed to unmapped stage
  • Salesperson reassigned to user not in mapping table
  • CompanyCam API temporarily down (retry logic)

Phase 4: Production Dry Run

  • Deploy to staging environment with copy of production data
  • Monitor for 48 hours with real triggers
  • Validate no duplicate syncs occur
  • Check error logs and notification alerts
  • Get client approval before production deployment

Phase 5: Go-Live Monitoring

  • Deploy to production with verbose logging enabled
  • Monitor first 100 syncs closely
  • Set up alerts for error rates >5%
  • Daily check-ins with client for first week
  • Gradual reduction of logging after stable week
